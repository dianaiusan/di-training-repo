# Event Page Generation

`docs/events/upcoming.md` and `docs/events/past-events.md` are generated by `scripts/generators/generate_events.py`.

Source data:
- All markdown files under `docs/all-training/adverts/`
- `_template.md` is ignored
- Only frontmatter is used

Required frontmatter fields:
- `title`
- `slug`
- `start_date`
@@
@@Optional frontmatter fields (with defaults):
@@- `end_date` (defaults to `start_date` if not specified)
@@
@@Additional optional frontmatter used in cards:
@@- `url` (preferred card link target; supports internal or external links)
@@- `dates` (array; use for display when an event has gaps between sessions)
- `format`
- `level`
- `short_description`
- `tags`

Classification rules:
- ongoing: today is between `start_date` and `end_date`
- upcoming: `start_date` is after today
- past: `end_date` is before today

Output:
- `docs/events/upcoming.md` gets ongoing and upcoming sections
- `docs/events/past-events.md` gets past events

Link resolution:
- If `url` is set, it is used as the card link as-is.
- If `url` is empty, fallback is `/explore/training-catalogue/<slug>/`.

To regenerate:

```bash
.venv/bin/python scripts/generators/generate_events.py
```
