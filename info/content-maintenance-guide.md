# Content Maintenance Guide

This document is a practical playbook for adding and updating training content.

## 1. Add a new course

1. Copy `files/course-template.md` into `docs/explore/all-training/<slug>.md`.
2. Set required frontmatter fields:
   - `title`
   - `slug` (must match file name without `.md`)
  - `status` (`upcoming`, `ongoing`, `past`, `cancelled`)
   - `format` (`online`, `on-site`, `hybrid`, `self-study`)
   - `duration`
   - `short_description`
   - `tags`
   - `scientific_domains`
   - `learning_paths`
3. Keep a `**Dates**:` or `**Date**:` line in the body if dates are known.
4. Run:

```bash
source .venv/bin/activate
python3 scripts/generate_all.py
```

5. Check generated updates and verify links/status placement.

## 2. Add or change tags

Tag behavior is driven by frontmatter and mapping in `scripts/generate_tags.py`.

### Existing tag

- Add the tag slug in course frontmatter `tags`.
- Regenerate with `scripts/generate_all.py`.

### New tag

1. Add the slug to course frontmatter `tags`.
2. Add category mapping in `scripts/generate_tags.py` (`TAG_TO_CATEGORY`).
3. If needed, add/adjust category metadata (`CATEGORY_META`) and ordering (`CATEGORY_ORDER`).
4. Ensure badge style exists in `docs/assets/stylesheets/extra.css` for that category class.
5. Run full generation.

## 3. Add or change scientific domains

Domain pages are generated from frontmatter `scientific_domains` in course files.

1. Add domain slug to one or more course files under `scientific_domains`.
2. Run generation.
3. If a display label/icon is missing or wrong, update the domain metadata in `scripts/generate_scientific_domains.py`.

## 4. Add or change learning paths

Learning path pages are generated from definitions in `docs/explore/learning-paths/`.

1. Add/update a learning path definition markdown file in `docs/explore/learning-paths/`.
2. Use valid course slugs only.
3. Run generation. `validate_learning_paths.py` will fail on invalid references.
4. Add nav entry in `zensical.toml` if this is a new path that should appear in menu.

## 5. Grouping and listing styles

Shared CSS is in `docs/assets/stylesheets/extra.css`.

### Style families used for list/group views

- `ev-*`: Event cards and status badges (`upcoming`, `past`, `ongoing`, `cancelled`).
- `diff-*`: Difficulty/level badges (`beginner`, `intermediate`, `advanced`).
- `bd-*`: Category badges and bundle/list cards.
- `lp-*`: Learning path overview/detail layouts.
- `tg-*`: Tag category overview cards.
- `tag-*`: Tag detail cards and course links.
- `sd-*`: Scientific domain card layouts.
- `tag-list`: Course tag row container.

## 6. Color scheme customization

Color system is tokenized in `:root` at the top of `docs/assets/stylesheets/extra.css`.

Primary groups of tokens:

- Structural tokens:
  - `--clr-surface`, `--clr-border`, `--clr-border-light`
  - `--clr-shadow`, `--clr-shadow-lg`
- Text tokens:
  - `--clr-text-primary`, `--clr-text-secondary`, `--clr-text-muted`
- Accent tokens:
  - `--clr-accent`, `--clr-accent-bg`
- Category badge tokens:
  - `--clr-cat-hpc`, `--clr-cat-parallel`, `--clr-cat-gpu`, ...
- Difficulty tokens:
  - `--clr-diff-beginner`, `--clr-diff-intermediate`, `--clr-diff-advanced`
- Event status tokens:
  - `--clr-ev-upcoming`, `--clr-ev-ongoing`, `--clr-ev-past`, `--clr-ev-cancelled`

### Recommended change flow for colors

1. Update only token values in `:root` first.
2. Rebuild and preview.
3. Change per-class rules only when token-level changes are insufficient.

## 7. Status grouping rules for all-training catalog

`docs/explore/all-training/index.md` is generated by `scripts/generate_all_training_index.py` and grouped in order:

1. Upcoming
2. Past
3. Cancelled
4. Planned

The section mapping is controlled by `SECTION_STATUSES` in that script.

Important: `planned` is currently a section label in the generated page, not a validator-approved `status` value.

## 8. Common pitfalls

- Editing generated sections manually; changes will be overwritten.
- Slug mismatch between file name and frontmatter `slug`.
- Introducing new tag/domain/path slugs without updating relevant metadata in scripts.
- Skipping full generation after metadata changes.

## 9. Minimum verification before commit

```bash
source .venv/bin/activate
python3 scripts/generate_all.py
```

Then confirm:

- generation completed with no errors
- Zensical build succeeded
- internal link check succeeded
- generated diffs look intentional
