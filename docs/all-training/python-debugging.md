# Debugging and Profiling Python on HPC

## Summary

Python code that works on a laptop can fail silently or run slowly on an HPC cluster. This course covers systematic debugging with `pdb` and `ipdb`, profiling with `cProfile`, `line_profiler`, and `py-spy`, identifying memory issues, and understanding where time is actually spent in scientific Python code. Exercises run on real NAISS systems.

## Learning objectives

- Debug Python programs interactively and post-mortem
- Profile Python code to identify CPU and memory bottlenecks
- Interpret profiler output and prioritise optimisations
- Use logging effectively in batch HPC jobs
- Understand common HPC-specific failure modes in Python

## Prerequisites

- Python on HPC (python-hpc or equivalent)
