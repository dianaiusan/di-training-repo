# HPC Filesystems and Storage Management

## Summary

HPC systems provide several distinct storage areas — home directories, project storage, scratch space, and long-term archives — each with different performance characteristics, quotas, and retention policies. This course explains how to use each tier correctly, avoid common mistakes like filling scratch space, and design workflows that move data efficiently.

## Learning objectives

- Distinguish between home, project, scratch, and archive storage
- Understand quota limits and retention policies on NAISS systems
- Choose the right storage tier for different stages of a workflow
- Move and stage data efficiently between tiers
- Avoid common pitfalls that cause jobs to fail or data to be lost

## Prerequisites

- Basic Linux command line and HPC familiarity (naiss-intro or equivalent)
