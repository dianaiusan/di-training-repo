# Programming Formalisms

Developing academic software that you can trust to be 'good enough'.

## Course Details

**Dates**: 4 - 8 May 2026  
**Location**: Online  
**Duration**: 5 days  
**Level**: Intermediate

## Summary

This course focuses on developing reliable academic software. You'll learn formal methods and best practices for writing code that you can trust for research purposes.

## Learning objectives

- Understand software quality principles
- Learn testing methodologies
- Apply formal verification techniques
- Implement code review practices
- Use version control effectively

## Topics Covered

- Software testing (unit, integration, system)
- Code quality metrics
- Formal specification methods
- Version control best practices
- Code review techniques
- Documentation standards

## Prerequisites

- Experience programming in at least one language
- Basic understanding of software development

## Target Audience

- Researchers developing software for their work
- PhD students writing computational code
- Staff supporting research software development
